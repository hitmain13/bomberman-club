# Git (canonical)

- Conventional Commits.
- Mensagens em inglês.
- Tipos: `feat`, `fix`, `refactor`, `perf`, `docs`, `test`, `build`, `chore`, `ci`, `style`.
- Escopo recomendado entre parênteses (ex.: `feat(auth): add refresh token rotation`).
- Tag por milestone: `m1`, `m2`, ...
- Nunca commitar com testes falhando, typecheck quebrado ou warnings de console.
